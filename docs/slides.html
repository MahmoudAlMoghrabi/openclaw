Level 4 · Your turn

Build your own 🎨

Three ways in — pick whichever fits you:

🔥 Guided: code-roaster

  • Fill the 5 TODOs in skills/guided-skill/SKILL.md
  • Test on roast-me.js · answer key: SOLUTION.md

🧱 From scratch

  • skills/my-first-skill/SKILL.md
  • Name it, describe it, write the instructions

💡 Idea menu

  • skills/EXAMPLES.md + stretch-ideas.md
  • Excuse generator, pirate rewriter, quiz maker...

Whichever you pick: the description is the trigger (vague = never fires), and the loop is the same. Do not touch the --- lines.

Level 5 · Hands out

Make it act 🙌

So far your agent has only talked. Time for hands. Say: "Read repo/skills/guided-skill/roast-me.js yourself and roast it." It fetches the file — you pasted nothing.

Then: "Fix everything you roasted and save it as repo/roast-me.fixed.js — show me the plan first." Approve, then watch the file explorer: a new file appears that you didn't write.

Talking about code is a chatbot. Changing it is an agent.

Level 6 · Real tools

Call a real tool 🎲🌦️

Your agent ships with built-in tools — read, write, exec, web_fetch — and you can plug in new ones over MCP. Try one of each:

🌦️ Built-in: web_fetch

  • "what's the weather in St. John's right now?" — live data, fetched not guessed

🎲 Plugged-in: MCP

  • "roll 3 dice" — real randomness from mcp/workshop-tools.js, ~100 lines you can read

The model was trained months ago: it can't know today's weather or roll real dice. Its tools can. That's why agents beat chatbots.

Level 7 · Meta mode

Let the agent build one 🤯

In the chat, describe a skill you want, and add the magic words: "show it to me first — do not save any files until I approve."

Read its draft like a code review, demand one change, then approve — and watch skills/ in the file explorer: your agent writes its own new capability. The agent proposes, you decide.
